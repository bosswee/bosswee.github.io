由LeetCode SubSets问题引发的ArrayList深拷贝的问题

發表於 2015-04-15   |   分類於 LeetCode   |  

Subsets

Given a set of distinct integers, S, return all possible subsets.

Note:

  • Elements in a subset must be in non-descending order.
  • The solution set must not contain duplicate subsets.
    For example,
    If S = [1,2,3], a solution is:
    [
    [3],
    [1],
    [2],
    [1,2,3],
    [1,3],
    [2,3],
    [1,2],
    []
    ]
    一开始这么写：

[java]
public class SubSets {
private static List<List<Integer>> res = new ArrayList<List<Integer>>();
public static List<List<Integer>> subsets(int[] S) {
Arrays.sort(S);
ArrayList<Integer> tmpres = new ArrayList<Integer>();
dfs(S, 0, tmpres);
return res;
}

private static void dfs ( int[] S, int i, ArrayList<Integer > tmpres){
if (i == S.length) {

res.add(tmpres);//res的值好像会随着tmpres的值改变。。。似乎增加的只是tmpres的引用？
return;
}
tmpres.add(S[i]);
dfs(S,i+1,tmpres);

tmpres.remove(tmpres.size()-1);//tmpres remove为什么会把res的中的值改变？？
dfs(S,i+1,tmpres);//尼玛不知道哪错了
}

public static void main(String[] args) {
int[] test = {1,2};
System.out.print(subsets(test));
}

}
[/java]

结果报错，res的值全是[][][]…
step into看了一下发现res中tmpres的值居然会随着tmpres的值更改，也就是说res.add()只是添加了tmpres的引用，dfs部分修改如下。

[java]
private static void dfs ( int[] S, int i, ArrayList<Integer > tmpres){
if (i == S.length) {
List<Integer> tmpclone = new ArrayList<Integer>();
for (int j : tmpres) {
tmpclone.add(j);
}//深拷贝下
res.add(tmpclone);
return;
}
tmpres.add(S[i]);
dfs(S,i+1,tmpres);

tmpres.remove(tmpres.size()-1)；
dfs(S,i+1,tmpres)；
}
[/java]

或者

[java]
private static void dfs ( int[] S, int i, ArrayList<Integer > tmpres){
if (i == S.length) {
res.add((List<Integer>) tmpres.clone());//深拷贝
return;
}
tmpres.add(S[i]);
dfs(S,i+1,tmpres);

tmpres.remove(tmpres.size()-1);
dfs(S,i+1,tmpres);
}
[/java]

关于java深拷贝和浅拷贝的知识点：
预定义集合类型–clone方法：实际是逐个调用每个元素的operator=方法因此，预定义集合类型的clone()结果为浅拷贝还是深拷贝，取决于其元素类型
1.预定义集合类型,元素为预定义非集合类型,执行clone()方法 – 深拷贝
2.预定义集合类型,元素为预定义集合类型,执行clone()方法 – 实际是逐个调用每个元素的operator=方法，即这时的clone()方法其实为浅拷贝。比如二维链表…
