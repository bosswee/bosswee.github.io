leetcode习题分类汇总-Array篇

Posted on 2015-06-02   |   In LeetCode   |  


### 题记：刷了一大部分leetcode之后总想找个时间总结一下，也为了以后复习整理方便，这里给每个数据结构写一个分类汇总，总结遇到类似问题时的一般方法和思路。选取了62题中的45题。（其他有些未做暂时难度有点大，有些没意义）





###

### 一组：二分搜索相关：

###


1.Search Insert Position-查找数组中目标值指定位置，没有则插入，返回插入位置

思路：(1)遍历，大于等于话返回i.小的话往后直到结束返回i+1,时间复杂度O(n),空间O(1).(2)二分搜索，时间O(logn)



### 2.Find Minimum in Rotated Sorted Array-寻找旋转过后有序数组最小值

思路：二分搜索，只是方向判断的条件不同，
若中间数大于等于最左端数，那左半部分必定有序，否则右半部分有序。时间logN


### 3.Find Minimum in Rotated Sorted Array II-寻找可旋转数组的最小值，允许重复

思路：二分搜索，只是加个对左右两边的判断条件，如果相同则把其中一个去掉，做这种题最重要的就是多实验，注意细节。时间logn



### 4.Find Peak Element-寻找数组中比两边元素都大的元素，时间要求logn

思路：二分搜索。注意“首先我们找到中间节点mid，如果大于两边返回当前index就可以了，如果左边的节点比mid大，那么我们可以继续在左半区间查找，这里面一定存在一个peak”这个特性



### 5.Search in Rotated Sorted Array-寻找旋转数组中的目标值位置，无重复值

思路：还是和寻找最小值差不多，和两边的元素对比判断，只是又加了个对目标值的判断条件，二分搜索。时间logN.


6.Search for a Range-logN时间内寻找目标值在有序数组中的范围

思路：二分搜索找到第一个>=target的位置，然后在二分找到第一个>=target+1的值的位置(然后减去1），还是注意细节吧…



### 7.Median of Two Sorted Arrays-寻找两个排序数组的中位数，要求时间复杂度log(m+n)

思路：（1）归并排序的简版，注意m+n奇偶对中位数的影响就好，时间复杂度(m+n)/2

(2)二分搜索，把问题转变为寻找第k小数的问题，还是分奇偶讨论


总结：二分搜索非常常见，主要为了把时间复杂度提高到logn的级别，遇到在数组有序或可以构造有序条件时寻找目标值或者范围可以优先尝试二分搜索。







二组：DP相关：





1.Maximum Subarray-寻找和最大的连续子数组

思路：一维DP，维护一个最大值，遍历，不断计算包含i的前i个元素的最大值，避免前i-1最大值的重复计算，时间n,空间n



2. Maximum Product Subarray-求最大乘积的子数组

思路：（1）记录每种情况的最大值，O(N2)

(2)一维DP,注意和加法的不同，可能出现负数，所以在维护最大值的同时也要维护最小值，O(N).




3.Best Time to Buy and Sell Stock-一天中T+1买卖股票获得的最大利润

思路：一维DP,维护一个maxProfit和一个前i时间段内的最小值，可以复用前i-1时间段内最小值的重复计算，时间n；



4.Best Time to Buy and Sell Stock II-一天内T+0买卖股票获得的最大利润

思路：贪心，逆向遍历，就是只要有利润那就买卖，最终求出最大值，额，有技术含量吗…时间N



5.Minimum Path Sum-矩阵的最短路径问题

思路：经典二维DP,维护到当前步骤的最小值，复用，直到最后一个最小值出现。时间N.



6.Jump Game-测试数组是否可以从前跳跃到尾部

思路：贪心？..遍历，记录出每次跳跃的最大步骤max，其实只要知道什么时候false的条件就可以了,max<=i而且当前A[i]=0的时候，大概这种题主要就是考验细心吧，至于DP,在求max的时候要用到，贪心是DP的特殊情况，没看出来，容我日后补补。时间N




7.Jump Game Two- 测试数组从前跳跃到后面的最小步数

思路：
比较典型的贪心。维护一个区间，区间表示第i步所能到达的索引范围。递推的方法为：
每次都遍历一遍当前区间内的所有元素，从一个元素出发的最远可达距离是index+array[index]，那么下一个区间的左端点就是当前区间的右端点+1，下一个区间的右端点就是当前区间的max
(index+array[index])，以此类推，直到区间包含了终点，统计当前步数即可。


### 8.Unique Paths-mn个方格机器人从左上到右下一共多少种方案

思路：二维DP，构造二维数组，用数字表示走到当前格子有多少种方法，递推式就是当前格子等于左边加上上面格子的总方法，不断复用，遍历矩阵得到结果，时间n,空间n.


9. Unique Paths II-同上，假设有障碍物有多少种方案

思路：递推式还是跟UniquePaths一样，只是每次我们要判断一下是不是障碍，如果是，则res[i][j]=0，否则还是res[i][j]=res[i-1][j]+res[i][j-1]。还是没习惯一维数组的解法，感觉二维的更加直观。


总结：DP就更加常见了，利用目的通常是重复利用之前的问题减少计算量以减少时间复杂度，在遇到极值问题如最短路径，最大…以及多少种方案等问题时可以优先考虑。





三组：借助其他数据结构 ：






### 1.Contains Duplicate-判断数组中是否有重复值

思路：创建hashmap key对应值，然后遍历 时间复杂度O(n),空间复杂度O(n)




2.Longest Consecutive Sequence-O(n)时间内找到无序数组中的最长连续序列

思路：改变数据结构，换为HashSet(remove,add,contains操作均为O(1)),遍历数组，计算每个元素的左右边界（维护count)，并每次减去当前扩展值（防止重复计算，减少时间），最后得出max.


3.Two Sum-数组中寻找两个数之和等于目标值，假设有且只有一个解。
思路：（1）先排序，然后两个指针一个在左一个在右边，时间logN+N;

*(2)减少时间复杂度一般采用hashtable,这样可以省去排序的时间，key为target-当前值，value为当前i,如果碰到第二个合适的数就可以根据hash找到前一个数的索引然后存入int[2]里就可以了,时间N,空间N

四组：DFS相关：





1.SubSets-返回所有子数组，原数组无重复

思路：dfs，回溯，注意list的浅拷贝和深拷贝。



2.SubSets II -返回所有子数组，原数组有重复

思路：依然是dfs，回溯，只是要注意消除重复的情况，加到容器里的时候加个判断就可以了，时间n2




### 3.Combination Sum-给定目标值返回数组中所有=target值的排列组合，元素可以重复利用

思路：回溯，dfs,因为不清楚当前元素利用的次数，所以是NP问题。



4.Combination Sum II-给定目标值返回数组中所有=target值的排列组合，元素不能重复利用重复利用

思路：回溯，dfs,和前一题类似，只是返回i+1而已。全排列的问题好像都是一个模型..


5.Combination Sum III -给定元素个数从1-9中选取指定元素，求等于目标值的排列组合。

思路：依然是回溯，dfs，只不过传的参数多一点而已



6.Construct Binary Tree from Inorder and Postorder Traversal-已知中序和后序遍历构造BT

思路：利用中序遍历的左中右和post的左右中，不断切分，map建立索引，dfs可以得出当前节点的左右子树，相当于一次树的遍历，时间N,空间N;



### 7.Construct Binary Tree from Preorder and Inorder Traversal已知前序和中序遍历构造BT

思路：一样吧…利用中序遍历的左中右和pre的中左右，不断切分，map建立索引，dfs可以得出当前节点的左右子树，相当于一次树的遍历，时间N,空间N，这样的题目多练习就好。


*总结：DFS出现概率也很高，主要也就是不断递归求解，主要用于返回多种排列组合的问题求解。





五组：双…N指针

###


1.Sort Colors-排序颜色，就是自己排序包含012三个数字的数组给出一遍遍历原地排序的解法

思路：91）双指针，快速排序的思想，两头往中间遍历。一遍遍历

（2）两次遍历，第一次计算123的频率，第二次把123填入到数组里。



2.Container With Most Water-计算最大容水量

思路：双指针遍历，当从两边向中间考虑时，乘水的面积是由（两端较小的高度）×（两个板之间的宽度）决定的。想要变得大一点，只有增加高度以期望超过宽度的减少。所以维护一个maxarea,然后不断增加高度以减少宽度的减少。时间N



3.Remove Duplicates from Sorted Array-去除数组的重复值并返回长度，不借助其他数据结构

思路：注意已经排序，快慢指针，快指针判断，慢指针填充，如果要返回数组的话copy一个，返回长度的话就是慢指针+1；时间N


4. 3 Sum Closest-在数组中找到3个数字的和使其最靠近目标值

思路：一开始我以为用dfs，时间复杂度N2;后来发现用三个指针维护也行，当前，next，已经从最后一个元素开始向前的指针，时间复杂度貌似也是N2,两个内循环;



5.ThreeSum-数组中寻找三个数的和等于0

思路：这题就是比3sum closest 多考虑一个去重问题，因为排好序了所以也简单，1.利用两个指针寻找sum； 2.去重 时间N2




6.4Sum-数组中寻找4个数的组合之和等于目标值，要求无重复的列出
思路 ：跟3 sum差不多，只是需求扩展到四个的数字的和了。我们还是可以按照3 sum中的解法，只是在外面套一层循环，相当于求n次3 sum。我们知道3 sum的时间复杂度是O(n^2)，所以如果这样解的总时间复杂度是O(n^3)。

（2）hashtable,具体未展开了解…

扩展：k-sum问题





7.Minimum Size Subarray Sum-给定数组和目标值求长度最小且不小于目标值的左数组。

思路：(1)两个指针遍历数组，维护长度最小值，时间O(N);
** (2)NlogN解法，*构造一个递增序列以便二分搜索, 不展开了，感觉没啥意义


总结：多指针通常是为了减少空间复杂度的情况下遍历数据结构，通常为了解决一些动态变化，维护最值，或者排序时的问题。



  









六组：利用本身特性


### 


1.Majority Element-寻找数组中出现频率最大的数字，且假设频率大于n/2

思路：（1）利用特性直接排序然后返回2/n的数值。O(nlogn)

(2) 设定一个count值，维护一个major，最后返回出现频率最大的值，O(n)



2.Remove Element-去除数组中的目标元素并返回长度·
思路：额，遍历一遍，维护一个next,把返回长度的数组顺序弄好就ok了，感觉木有什么意义啊…好像就是考察你如何在去除的同时记录长度，时间n。






3.Remove Duplicates from Sorted ArrayII-去除数组两次以上的重复值并返回长度


思路：维护一个counter和 idx，>3时跳过，A[idx++]=A[i]







4.Rotate Image-旋转矩阵


思路：这个问题还是有固定套路的，和玩魔方一样，上面和最后一行互换，然后对角线元素互换，还是比较巧妙的，私以为这种有现成方案的问题还是知其然就好，但是不同语言的表达能力不同，所以这题最好用c++方便点，因为有swap函数






5.Plus One-模拟加法，数组表示数字加一然后返回


思路：纯数学，注意进位carry和特殊情况999+1…





6.Merge Sorted Array-合并两个有序数组,第一个数组有足够空间
思路：归并排序，只不过，因为防止元素被破坏逆向遍历。时间m+n









7.First Missing Positive-找到无需数组中缺失的正整数 常量空间与时间


思路：常量时间的排序不基于比较，这题目就是桶排序的变种，利用数组的下标这样就可以不用hash table记录了，目标是尽可能将数字i放到数组第i-1个位置。


扫描数组中每个数：
1. 如果A[i]<1或者A[i]>n。说明A[i]一定不是first missing positive。跳过
2. 如果A[i] = i+1，说明A[i]已经在正确的位置，跳过
3. 如果A[i]!=i+1，且0<A[i]<=n，应当将A[i]放到A[A[i]-1]的位置，所以可以交换两数。
这里注意，当A[i] = A[A[i]-1]时会陷入死循环。这种情况下直接跳过。




8.Rotate Array-给出旋转个数旋转数组


思路：三次旋转，主要就是数组翻转的基本操作，比如k=3,n=9,1234567,4321，765，然后5671234。和矩阵链表的翻转一样，没啥区别…


 


 9. Spiral Matrix II -给定n,返回二维数组表示的螺旋矩阵
 思路：递归构造，时间N2，不谈了，还是多练


 




10.Search a 2D Matrix-在一个按顺序递归的螺旋矩阵中寻找目标值


思路：遍历就不说了，减少时间复杂度肯定用二分搜索


（1）做两次二分就好了，首先二分第一列，找出target所在的行，然后二分该行。


（2）利用矩阵的特性


int mid_value = matrix[mid / col_num][mid % col_num];

，把它看做一个有序数组，一次二分即可


 


11.Set Matrix Zeroes-在矩阵中把0的行列全部设为0；


思路：


check if first row and column are zero or not
mark zeros on first row and column
use mark to set elements
set first column and row by using marks in step 









总结：每种数据结构都有自身的特性，根据题意也有不同的效果，比如数组可以利用自身下标，矩阵可以看做有序数组等等，这些隐含条件往往在解题时会有意想不到的效果。








 


后记：总结主要为了以后遇到问题时的思路可以更加清晰，而不是干是看着问题，却无法下手，有一个清晰的逻辑过程，并且在实践中不断完善这个过程是练习的最终目的。简单说下解题的步骤：（我擦ヾ(｡｀Д´｡)，感觉回到高中时代了…)


1.拿到题目先考察题目要求,隐含条件


2.直觉给出第一方案（长期练习习得）


3.考虑其他方案的可行性，排除法


4.演算，并给出最终方案


5.编码


6.复查，测试
